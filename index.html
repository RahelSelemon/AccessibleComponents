Accessible Components Case Study

I explored dropdown menus—one of the most widely used UI components—and found significant accessibility issues across platforms. This case study walks through my analysis and redesign process to make dropdowns more inclusive and intuitive.

Why Dropdowns Deserve a Rethink

Dropdown menus are commonly used in web and app interfaces to group multiple options into a single element. However, they often fall short in accessibility, especially for screen reader users and keyboard navigators. I wanted to explore how small changes in structure and feedback could improve this core component for all users.

Who I Designed This For

This project focuses on improving the experience of:

  • Mouse users navigating visually
  • Keyboard users relying on focus order
  • Screen reader users needing semantic clarity
I wanted to ensure that the redesign was usable without sacrificing visual aesthetics or familiarity.

What is a Dropdown Menu?

A dropdown menu is a compact UI element that reveals a list of options. It helps reduce visual clutter and organizes choices efficiently. Common uses include navigation menus, settings, filters, and form selections.

Case Focus: Spotify

Spotify’s dropdown is well-placed and visually clear. It responds to both mouse and keyboard users effectively, and groups options into logical categories.

  • Hover feedback is immediate and helpful
  • Logical menu placement between album title and track list
  • Icons are clear and navigation is cyclical with keyboard

Opportunities for Improvement

Despite strengths, there were accessibility gaps, especially in screen reader structure and keyboard flow. I created revised state diagrams to address these issues.

Design & Prototype

Key Improvements

  • Improved feedback: Active icon state is clearly visible
  • Better discoverability: Icons enlarge on hover and have more spacing
  • Accessible layout: Logical groupings and screen reader-friendly accordion behavior

Takeaways

  • Mouse users are often prioritized in dropdown design, but keyboard and screen reader experiences suffer as a result.
  • Improving focus order and menu structure made navigation significantly more intuitive.
  • Subtle enhancements—like active outlines, larger icons, and grouped actions—had a big impact on usability.
  • I learned that VoiceOver can be overwhelmed by poor structure; thoughtful spacing and sectioning matter a lot.
  • Testing across multiple input modes (mouse, keyboard, screen reader) is essential for inclusive design.
